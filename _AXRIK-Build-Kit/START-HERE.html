HOW TO USE THE KIT

Set it up once. Then a short checklist each client.

You did the hard part once, on JG Foods. This kit means you never repeat that part. Two stages: set it up once, then run a quick checklist for every new client.

1

One-time setup

Do this once · about 15 minutes

Install the three skills

In the install-skills/ folder are three files ending in .skill. When I show them in chat, each has a Save skill button — click all three. Or go to Settings → Capabilities → add skill and pick each file:

axrik-project-kickoff — the build workflow · axrik-ai-features — AI features & fallbacks · axrik-deliverables — all the client paperwork.

Once installed they trigger automatically. From then on you just say "new AXRIK client" — you don't have to remember any of this.

Fold the Supabase additions into your existing skill

Open skills/supabase-patterns-ADDENDUM.md and copy it into your existing supabase-patterns skill (Settings → Capabilities → edit). That's the day-one RLS, the trigger fixes and the self-service-users pattern.

Park the starter-kit repo where you can clone it

The starter-kit/ folder is a git repo. Push it to GitHub as a template repo so you can clone it per client:

# from inside the starter-kit folder
git remote add origin git@github-axrik:AXRIK-dev/axrik-starter-kit.git
git push -u origin main

Then on GitHub: repo Settings → Template repository → tick it.

Move the whole kit out of the JG Foods folder

Right now _AXRIK-Build-Kit/ lives inside the JG Foods project. Move it to your shared AXRIK folder so it isn't tied to one client.

2

Each new client

The actual time-saver · you mostly just talk to me

Kick off — discovery first

In a new project for the client, say:

New AXRIK client: [name], a [type of business] in [location]. Here are their order messages / spreadsheets / invoice format [attach them]. Follow the kickoff workflow — propose the data model first, then the build phases. Don't write screens until I've confirmed the schema.

That's prompt #1 in the Prompt Library. The kickoff skill does the rest.

Clone the backend

# create the new repo from the template, then:
git clone git@github-axrik:AXRIK-dev/[client]-app.git
cp config.example.js config.js   # add Supabase URL + anon key + branding

Tell me the client's delivery days, order channels and whether they invoice — I customise the three SQL files (the >>> markers). You run them in Supabase in order 001 → 002 → 003, then run the "promote owner to admin" snippet at the bottom of 002.

Build on the shells

The website/ and admin/ shells already do auth, the role gate, the live catalogue and ordering. Add features by saying "add a [feature] page to the admin" (prompt #4). AI features come last, each with a fallback (prompt #5).

Go live & hand over

[Client] is ready for go-live. Produce the go-live checklist, the testing worksheet, and refresh the build summary.

Prompt #7. The deliverables skill produces all of it in AXRIK navy/gold, in your voice.

End every session with the hygiene sweep

Before we finish: any leftover placeholders, any table missing RLS, any AI button without a fallback, what's still stubbed?

Prompt #9 — this kills the "fix" commits that ate time on JG Foods.

The one-line version

Set up once: install 3 skills, fold in the Supabase addendum, push the starter kit as a GitHub template.

Each client: say "new AXRIK client", confirm the schema, clone the template, build on the shells, hand over from templates. Target ~15–18h instead of 35.
